# Feishu-Job-Radar (飞书求职雷达)

**Feishu-Job-Radar** 是一款专为校招、实习求职者打造的轻量级、全自动岗位监控与管理系统。它可以自动扫描抓取优质岗位信息，按照你高度定制的求职偏好进行个性化匹配，将筛选通过的优质岗位一键推送到你的**飞书多维表格（Bitable）**中，并支持**每日飞书群机器人卡片提醒**。

---

## 🌟 核心功能

* ⚡ **增量自动抓取**：定时自动运行爬虫扫描源网站最新岗位，并具备“全页面重复”动态停止机制，高效且省流。
* 🎯 **个性化规则匹配**：
  * **高灵敏推荐**：支持设置“必看公司”一键直推；支持设定目标岗位方向（如嵌入式、算法、软件开发）进行关键词打分。
  * **多层级过滤**：支持按目标城市过滤、自动比对公告届别（如必须招 26/27 届）和批次（秋招/实习等）。
  * **一票否决制**：强排除负向词（如运营、销售、HR、市场），杜绝非技术及无关岗位的干扰。
* 🔄 **双向自愈同步 (状态保护)**：
  * 本地数据库会自动检测并拉取你在飞书多维表格中手动标记的“用户状态”（如“已收藏”）和填写的“备注”。
  * 当你修改了本地匹配规则重新运行匹配时，**系统会自动保护你已收藏的状态和备注不被覆盖**，同时增量刷新飞书记录，达到云端和本地的安全一致性。
* 🤖 **飞书机器人卡片推送**：每日自动汇总匹配出的新增岗位，生成直观的卡片消息推送到群聊，点击直接打开岗位网申地址。
* 📦 **开箱即用（免零初始化）**：项目内置了清洗好个人隐私的种子数据库，用户初次启动即可一键继承历史已抓取的数千条优质岗位信息，直接重匹配出心仪职位！

## 🛠️ 部署与安装指南

### 1. 环境准备
确保你的本地安装了 **Python 3.11 或更高版本**。

### 2. 获取代码与依赖安装
1. 下载/克隆本项目到本地。
2. 在项目根目录下打开终端，创建 Python 虚拟环境：
   ```powershell
   python -m venv .venv
   ```
3. 激活虚拟环境：
   * **Windows (PowerShell)**: `.\.venv\Scripts\Activate.ps1`
   * **Windows (Cmd)**: `.\.venv\Scripts\activate.bat`
   * **Linux/macOS**: `source .venv/bin/activate`
4. 安装项目及其依赖：
   ```powershell
   pip install -e .
   ```

### 3. 创建配置文件
1. 将根目录下的 `config.example.yaml` 复制一份并重命名为 `config.yaml`：
   ```powershell
   copy config.example.yaml config.yaml
   ```
2. 打开 `config.yaml`，根据你的求职需求修改 `user_profile`：
   * **`graduate_years`**：你想投递的毕业届别（如 `"2026届"`, `"2027届"`）。
   * **`batches`**：求职类型批次（如 `"秋招"`, `"实习"`, `"提前批"`）。
   * **`target_cities`**：你想去的城市列表，留空表示不限城市。
   * **`role_groups`**：目标岗位方向。
   * **`must_watch_companies`**：你的必看大厂/目标企业清单。

## 🔗 飞书对接配置步骤

为了实现飞书看板同步与每日提醒，你需要完成以下对接配置并填入 `config.yaml` 中的 `feishu` 段落：

### 第一步：创建飞书自建应用
1. 登录 [飞书开放平台](https://open.feishu.cn/)。
2. 创建一个“企业自建应用”，并发布该应用（或在版本管理中提交一个初版进行发布）。
3. 获得应用的 **`App ID`** 和 **`App Secret`**。
4. 在“开发配置 -> 权限管理”中，搜索并开启 **“多维表格”的读取和写入权限**。

### 第二步：创建飞书多维表格看板
1. 在飞书中创建或打开一个多维表格。
2. 从浏览器地址栏提取 **`App Token`**（URL 中 `apps/` 后面的一长串字符）和 **`Table ID`**（URL 中 `table=` 后面的一长串字符）。
3. 将多维表格的列名配置如下（以便数据能正常同步写入）：
   * `岗位ID`（单行文本）
   * `公司`（单行文本）
   * `岗位名称`（单行文本，原 clean_title 映射）
   * `公告摘要`（多行文本）
   * `城市`（单行文本）
   * `批次`（单行文本）
   * `届别`（单行文本）
   * `收录日期`（日期类型）
   * `原始链接`（超链接类型）
   * `官方链接`（超链接类型）
   * `推荐状态`（单选）
   * `核验状态`（单选）
   * `用户状态`（单选，包含选项：`未看`、`已收藏`、`已网申` 等，**用于在飞书上手动勾选**）
   * `备注`（单行文本或多行文本，**用于在飞书上手写备注**）

### 第三步：配置群消息 Webhook 机器人（可选）
1. 在接收提醒的飞书群聊中，“设置 -> 群机器人 -> 添加机器人”，选择“自定义群聊机器人”。
2. 复制生成的 **`Webhook URL`**。

> ⚠️ 完成上述步骤后，将这 5 个飞书凭证项填入你本地的 `config.yaml` 对应位置。

## 🚀 常用运行命令

### 1. 每日扫描与飞书推送 (Standard Run)
自动进行新岗位的增量扫描、拉取飞书最新收藏状态与备注、向飞书推送新推荐、并向群聊发送每日简报：
* **命令行运行**：
  ```powershell
  $env:PYTHONPATH="src"; python -m job_monitor daily
  ```
* **一键运行**：在 Windows 上，直接双击运行根目录下的 **`run_daily.bat`** 即可。

### 2. 匹配规则修改后重新匹配 (Rematch)
如果你修改了 `config.yaml` 里的求职偏好，想要对历史数据库中的所有岗位按新规则重新推荐与飞书刷新（此过程会自动读取并保护飞书上已收藏的状态）：
```powershell
$env:PYTHONPATH="src"; python -m job_monitor rematch
```

### 3. 数据备份与离线导出
如果想将本地数据库的岗位清单与你的收藏状态导出为 Excel 格式进行本地离线归档：
```powershell
$env:PYTHONPATH="src"; python -m job_monitor export --table all --output data/exports/all_jobs.xlsx
```
*(注：导出的 Excel 文件将自动生成在 `data/exports/` 目录下)*
