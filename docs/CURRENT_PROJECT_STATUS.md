# 当前项目状态

更新时间：2026-07-11

## 已完成

- 面向个人求职者的本地 MVP 已完成，主要支持 Windows 10/11 和 Python 3.11/3.12。
- `init` 可生成最小配置，并在用户指定的飞书 Base 中自动创建“求职工作台”。
- 工作台包含 9 个用户字段、8 个求职状态，以及“待处理、收藏、投递进度”三个视图；公司为主列。
- `daily`、`rematch`、`pull`、`check`、`reset` 已形成日常使用、测试重建和排障闭环。
- 只同步推荐岗位和持续跟踪岗位；系统不会覆盖求职状态和备注，内部字段仅保存于本地。
- 未知远端记录、非法状态和回拉失败会停止同步，并返回明确错误。
- 飞书认证、读取和批量写入支持有界重试；重复初始化不会重复创建结构或岗位。
- README、示例配置、CI、MIT License 和 Windows 自动运行说明已补齐。

## 验证结果

- 自动化测试：`152 passed`。
- 全新 Windows 虚拟环境安装和测试通过。
- 自动化测试覆盖 9 字段工作台、画册收藏视图、投递进度看板及显式 reset。
- 用户字段回拉成功；更新系统字段后，求职状态和备注保持不变。

## 当前限制

- 岗位来源目前以 WonderCV 为主，页面结构变化可能影响抓取。
- 投递进度使用飞书默认看板卡片布局，按求职状态分组并排除“不合适”。
- 旧个人表迁移不是公共功能；历史备份仍保留，但未自动覆盖当前工作台数据。
- 暂不支持 OAuth、SaaS、多用户、网页配置和多数据源聚合。
