# 飞书向导截图采集清单

将原始截图放到 `docs/feishu-guide/raw/`。推荐使用 1440×900 或更高的桌面窗口，保持浏览器缩放 100%。不要提交未脱敏原图；处理完成并核对后，应从 Git 工作区外安全删除包含真实信息的副本。

| 文件名 | 打开与操作 | 截图必须包含 | 必须打码 |
| --- | --- | --- | --- |
| `1.png` | 打开 `https://open.feishu.cn/app` | 应用列表、创建企业自建应用 | 企业名、姓名、头像 |
| `2-1.png`、`2-2.png` | 创建企业自建应用 | 创建按钮、名称、描述 | 企业名、姓名、头像、已有应用名 |
| `3-1.png`、`3-2.png` | 进入“凭证与基础信息” | 导航入口、App ID、App Secret | App ID、App Secret、企业名、头像；Secret 必须实色完全遮盖 |
| `4-1.png`、`4-2.png`、`4-3.png` | 进入“权限管理”并搜索 `bitable:app` | 权限入口、开通按钮、权限项、确认按钮 | 企业名、头像、无关权限 |
| `5-1.png`、`5-2.png` | 打开“版本管理与发布”并确认发布 | 更新说明、保存、确认发布 | 企业名、姓名、头像 |
| `6-1.png`、`6-2.png`、`6-3.png`、`6-4.png` | 打开飞书多维表格并新建空白表格 | 立即体验、多维表格入口、新建空白多维表格、最终空白表格 | 文件名、企业名、头像 |
| `7-1.png`、`7-2.png`、`7-3.png` | 在 Base 中添加 JobPicky 文档应用并设为可管理 | 更多、添加文档应用、JobPicky、可管理、添加 | Base 名称、表格内容、企业名、头像 |
| `8.png` | 打开测试 Base 并聚焦浏览器地址栏 | 含 `/base/` 的地址 | Base token、文件名、表格内容、企业名、头像 |

采集时只使用专门的测试应用与空白测试 Base。不要使用生产岗位、真实用户表格或现有 JobPicky 配置。处理命令：

```powershell
python scripts/annotate_feishu_guide.py
```

脚本从清单读取裁剪、红框、编号和脱敏配置，输出到 `src/jobpicky/web/static/images/feishu-guide/`，不会覆盖 `raw/` 原图。逐张检查文字可读性、遮盖不可恢复、红框不挡按钮文字，再把对应 `captureStatus` 改为 `captured-redacted-reviewed`。
