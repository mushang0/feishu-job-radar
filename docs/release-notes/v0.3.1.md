# v0.3.1

## 修复内容

- 修复首次配置后启动扫描时，任务快照文件被 Windows 临时占用会导致后台任务永久停留在“准备扫描”的问题。
- 快照替换失败时增加短暂重试和错误日志，并确保任务始终清理活动状态；快照异常不再中断实际扫描。
- 修复首次全量匹配产生的推荐未计入“新增推荐”的问题；现在按扫描前后的推荐集合差异统计。
- 移除没有明确用途的“本次新增”统计，岗位雷达保留“新增推荐”“推荐岗位”“即将截止”。
- 统一使用网页收录日期 `collected_date` 进行最近收录排序、日期筛选、详情展示和导出，不再使用本地 `first_seen` 代替。
- 收录日期缺失时不再从岗位标题或推荐日期猜测、回填日期。

- 桌面端和移动端岗位雷达视觉检查通过。
